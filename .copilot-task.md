## Tech-implementation rules

- 用 Astro components(`.astro` files)。需要轻交互(blink eyes / 表单提交 / lang switcher)的地方写 inline `<script>` 即可,不引 React。
- Tailwind 用 `@astrojs/tailwind` 集成,在 `tailwind.config.mjs` 里设 `theme.extend.colors` 把上面的 token 暴露成 `bg-mochi-pink` 这种语义类。
- Phone mockup 和 Mochi SVG 抽成 `<PhoneMockup.astro>` 单组件,接受 `expression` prop(默认 `happy`)。
- BYOK 的兼容 logo 墙用 SVG inline(去 simple-icons CDN 下载或写简化版),避免外链 PNG 增加 LCP。
- Waitlist 表单:client-side fetch GitHub Issues API,需要用户自己的 token? 不,改用 GitHub 公共 issue 创建是不可能匿名的。**改方案**: 把表单 action 指向一个 `mailto:` 或先用 Formspree 免费档(50 submissions/月)。在代码里留 TODO 注释。
